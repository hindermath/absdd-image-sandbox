# Agent Session Logs

Diese Dateien sind historische Audit-Hinweise fuer agentische Arbeiten im
Repository. Sie dokumentieren, welche Aufgaben, Pruefungen, Eskalationen und
Grenzen in einzelnen Sitzungen sichtbar waren.

Die Session-Logs sind keine aktuelle Governance-Quelle. Verbindliche aktuelle
Regeln stehen in:

- `AGENTS.md`, `CLAUDE.md`, `GEMINI.md` und `.github/copilot-instructions.md`
- `COMPLIANCE-PLAN_RL-SE-001.md`
- `docs/secure-development/`
- den jeweils aktiven Spec-Kit-Artefakten unter `specs/`

Private Anbieter-, Konto-, Hostpfad-, Portal- oder Managementsystem-Bezuege
werden in Logs neutralisiert, wenn die Logs in der Praxis als aktuelle
Orientierung gelesen werden koennen. Projektbezogene Evidenz bleibt im
Repository erhalten, aber ohne konkrete private Infrastruktur als
Pflichtvoraussetzung.

---

These files are historical audit notes for agentic work in this repository.
They document tasks, checks, escalations, and boundaries that were visible in
individual sessions.

Session logs are not the current governance source. Current binding rules live
in:

- `AGENTS.md`, `CLAUDE.md`, `GEMINI.md`, and `.github/copilot-instructions.md`
- `COMPLIANCE-PLAN_RL-SE-001.md`
- `docs/secure-development/`
- the active Spec Kit artefacts under `specs/`

Private provider, account, host-path, portal, or management-system references
are neutralized in logs when the logs may be read as current guidance in
practice. Project evidence stays in the repository, but without concrete
private infrastructure as a mandatory prerequisite.
